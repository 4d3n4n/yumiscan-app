# v1.5.1 — Official Update

Date: 2026-03-14

## But

Document officiel unique de la passe `1.5.1`.

Objectifs:
- stabiliser l'experience PWA/mobile
- corriger les flux Auth encore fragiles
- fiabiliser dashboard, scans et miniatures
- mieux integrer l'Assistant IA dans le cycle de scan

## Correctifs principaux livres

### PWA / navigation

- `AppNavigation` a ete centralisee dans le layout principal

- le changement d'email a ete durci cote front et cote template

### Dashboard / scans

- un scan nouvellement lance apparait sans latence visible dans le dashboard
- un scan supprime disparait tout de suite sans `F5`
- les scans termines se revalident mieux au retour sur le dashboard
- la double pulsation des miniatures a ete reduite

### Pipeline image

- une preview storage dediee est produite cote front
- les nouvelles images stockees pour l'UI passent en preview compressee
- l'extension d'upload est derivee du MIME reel au lieu d'etre forcee

### Assistant IA

- l'Assistant IA fonctionne pour les scans `ambiguous` et `contains_allergen`
- le copy et le sheet ont ete epures
- les allergenes affiches reutilisent les pills partagees de l'app
- le retour Assistant IA est maintenant cache en base par scan et par langue
- `Regenerer` force un nouvel appel et remplace le cache courant

### Onboarding / UI mobile

- swipe tactile ajoute sur mobile
- simplification de la navigation mobile de l'onboarding
- slide Assistant IA rendue plus lisible
- fond onboarding adouci avec un voile haut blanc

### Donnees / BO

- `user_purchases` est conserve apres suppression utilisateur
- les suppressions de compte et de scan nettoient le storage reference

## Choix techniques a retenir

### Assistant IA

Le retour Assistant IA n'est pas stocke dans `result_json`.

Il est stocke separement dans `scans.assistant_cache_json` pour garder:
- `result_json` comme sortie deterministe du pipeline scan
- l'Assistant IA comme couche derivee, versionnable et cachee

Le cache est:
- par `scan_id`
- par langue (`fr` / `en`)
- invalide automatiquement si la signature source du scan change

### Scan sans allergene selectionne

Si aucun allergene / critere n'est selectionne:
- `Phase 2` n'est plus appelee dans le pipeline principal
- les ingredients non mappes restent en `ok`
- un scan ne remonte donc plus en `douteux` sans profil actif
