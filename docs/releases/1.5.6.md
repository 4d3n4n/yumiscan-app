# v1.5.6 — Refactor et hardening final Assistant IA

Date: 2026-03-17

Status: implementation locale

## Resume

Cette passe cloture le chantier Assistant IA ouvert en `1.5.5` avec un durcissement final du backend.

Objectifs atteints:

- refactor modulaire de `scan-ambiguous-assistant`
- prompts Gemini plus stricts avec exemple JSON explicite
- seconde passe de reparation JSON quand Gemini repond avec une structure invalide
- fallbacks deterministes conserves uniquement en dernier recours
- docs alignees avec le flux reel de production

## Changements principaux

### 1. Edge Function plus propre

La logique de `scan-ambiguous-assistant` est maintenant separee entre:

- `index.ts`
- `context.ts`
- `cache.ts`
- `helpers.ts`
- `fallbacks.ts`
- `gemini.ts`
- `schema.ts`
- `types.ts`

Impact:

- handler HTTP plus lisible
- responsabilites metier mieux isolees
- maintenance et tests facilites

### 2. Generation Gemini plus robuste

Le flux Assistant IA suit maintenant ce chemin:

1. prompt contraint avec exemple JSON attendu
2. validation stricte du JSON
3. passe de repair JSON via Gemini si la premiere sortie est invalide
4. sanitization backend
5. fallback deterministe seulement en dernier recours

Impact:

- moins de phrases generiques repetitives
- moins de dependance aux fallbacks
- meilleures chances d'obtenir une fiche naturelle et coherentement structuree

### 3. Fallbacks clarifies

Les phrases deterministes restent presentes, mais leur role est explicite:

- ce ne sont pas des textes front par defaut
- elles servent a garantir une reponse exploitable en cas d'echec modele
- elles sont centralisees dans `fallbacks.ts`
